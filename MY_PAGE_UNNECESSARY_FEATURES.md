# 마이페이지 불필요한 기능 검토 결과

## ❌ 제거 권장 기능

### 1. 프로필 수정 페이지 - 스킬 셋 섹션
**위치**: `src/app/(home)/my/update/ProfileUpdateClient.tsx`
- **문제점**: 
  - 스킬을 입력하고 추가할 수 있지만 실제로 저장되지 않음
  - "스킬 셋은 추후 데이터베이스에 저장될 예정입니다"라는 메시지만 표시
  - `formData.skills`는 상태로만 관리되고 API 호출에 포함되지 않음
- **권장 조치**: 
  - 스킬 셋 섹션 전체 제거
  - 또는 실제 저장 기능 구현

### 2. 프로필 수정 페이지 - 포트폴리오 파일 첨부 섹션
**위치**: `src/app/(home)/my/update/ProfileUpdateClient.tsx`
- **문제점**: 
  - "포트폴리오 파일 첨부 기능은 추후 구현 예정입니다"라는 메시지만 표시
  - 실제 기능이 전혀 없음
  - 포트폴리오는 이미 `/my/profile/portfolio` 페이지에서 별도로 관리됨
- **권장 조치**: 
  - 포트폴리오 파일 첨부 섹션 제거
  - 포트폴리오 관리는 별도 페이지에서만 처리

### 3. 팀 제안 페이지 - 사용되지 않는 함수
**위치**: `src/app/(home)/my/team-proposal/TeamProposalClient.tsx`
- **문제점**: 
  - `handleProposalSubmit` 함수가 정의되어 있지만 사용되지 않음
  - `ProposalDialog` 컴포넌트가 자체적으로 제안을 처리함
- **권장 조치**: 
  - `handleProposalSubmit` 함수 제거

### 4. 팀 제안 페이지 - 사용되지 않는 상태 필터
**위치**: `src/app/(home)/my/team-proposal/TeamProposalClient.tsx`
- **문제점**: 
  - `selectedStatuses` 상태와 필터 UI가 있지만 실제로 북마크 필터링에 사용되지 않음
  - `fetchBookmarkList` 호출 시 `isProposed: false`만 사용하고 상태 필터는 무시됨
- **권장 조치**: 
  - `selectedStatuses` 상태 및 관련 UI 제거
  - 또는 실제 필터링 기능 구현

### 5. 관심 메이커 페이지 - 빈 함수
**위치**: `src/app/(home)/my/bookmarked-makers/BookMarkedMakerClient.tsx`
- **문제점**: 
  - `handlePropose` 함수가 비어있고 사용되지 않음
  - `ManageableMakerCard`에 전달되지만 실제 동작 없음
- **권장 조치**: 
  - `handlePropose` 함수 제거 또는 구현
  - `ManageableMakerCard`에서 `onPropose` prop 제거 (사용하지 않는 경우)

### 6. 사이드 네비게이터 - 존재하지 않는 페이지 링크
**위치**: `src/components/SideNavigator.jsx`
- **문제점**: 
  - `/my/counsel` 경로가 메뉴에 있지만 실제 페이지가 없음
  - `/my/proposal` 경로가 메뉴에 있지만 실제 페이지가 없음
- **권장 조치**: 
  - 해당 메뉴 항목 제거
  - 또는 실제 페이지 생성

### 7. 빈 디렉토리
**위치**: 
- `src/app/(home)/my/projects/` - 빈 디렉토리
- `src/app/(home)/my/team-profile/` - 빈 디렉토리
- **권장 조치**: 
  - 사용하지 않는다면 디렉토리 제거
  - 또는 실제 페이지 구현

## ⚠️ 개선 권장 사항

### 1. 프로필 수정 페이지 - 스킬 셋
- **옵션 1**: 완전히 제거 (가장 간단)
- **옵션 2**: 실제 저장 기능 구현 (DB 스키마 추가 필요)

### 2. 팀 제안 페이지 - 상태 필터
- **옵션 1**: 필터 UI 제거 (현재는 사용되지 않음)
- **옵션 2**: 실제 필터링 기능 구현 (API 수정 필요)

### 3. 관심 메이커 페이지 - 제안 기능
- **옵션 1**: 제안 기능 제거 (팀 제안 페이지에서만 사용)
- **옵션 2**: 제안 기능 구현 (팀 제안 페이지와 동일한 로직)

## 📋 우선순위별 제거 권장

### 높은 우선순위 (즉시 제거 권장)
1. **프로필 수정 - 포트폴리오 파일 첨부 섹션** - 완전히 불필요
2. **사이드 네비게이터 - 존재하지 않는 페이지 링크** - 404 에러 발생 가능
3. **팀 제안 페이지 - 사용되지 않는 함수** - 코드 정리

### 중간 우선순위
4. **프로필 수정 - 스킬 셋 섹션** - 저장되지 않는 기능
5. **팀 제안 페이지 - 사용되지 않는 상태 필터** - 혼란 야기 가능

### 낮은 우선순위
6. **관심 메이커 페이지 - 빈 함수** - 기능적 문제는 없음
7. **빈 디렉토리** - 기능적 문제는 없음

## 🔍 추가 확인 필요

1. **프로필 수정 페이지의 스킬 셋**
   - 향후 스킬 기능이 필요할지 확인
   - 필요하다면 DB 스키마 설계 후 구현

2. **팀 제안 페이지의 상태 필터**
   - 제안 상태별 필터링이 필요한지 확인
   - 필요하다면 API 수정 후 구현

3. **관심 메이커 vs 팀 제안 페이지**
   - 두 페이지의 역할이 중복되는지 확인
   - 통합 가능 여부 검토
