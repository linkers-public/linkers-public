# 기업 헤더 - 내 프로젝트 목록 기능 요구사항

## 개요
기업 헤더의 "내 프로젝트 목록" 페이지에 필요한 기능들을 정리한 문서입니다.

## 필터 카테고리

### 1. 전체 (All)
- **기능**: 상세 조회
- **내용**: 
  - 매칭중인 프로젝트 리스트
  - 견적 도착 프로젝트 리스트
  - 계약 진행 프로젝트 리스트
- **현재 상태**: ✅ 구현됨

### 2. 매칭중 (Matching) - `recruiting`
- **기능 1**: 매칭중인 프로젝트 상세 조회
  - 매칭중인 프로젝트 리스트 표시
  - 프로젝트 클릭 시 상세 페이지로 이동
- **기능 2**: 프로젝트 수정 ⚠️ **필요**
  - 기업에서 올린 프로젝트 견적 내용 중 수정 사항
  - 매칭중 상태일 때만 수정 가능
  - 수정 버튼/기능 추가 필요
- **현재 상태**: 
  - ✅ 상세 조회 구현됨
  - ❌ 프로젝트 수정 기능 없음

### 3. 견적 도착 (Quote Arrived) - `estimate_received`
- **기능**: 견적 상세 조회
  - 견적 도착한 프로젝트 리스트 표시
  - "견적서 검토" 버튼으로 견적 상세 페이지 이동
- **현재 상태**: ✅ 구현됨 (견적서 검토 버튼 있음)

### 4. 계약 진행 (Contract in Progress) - `contract_progress`
- **기능**: 계약 진행 중인 프로젝트 리스트 표시
  - 계약 진행 중인 프로젝트 상세 조회
- **현재 상태**: ✅ 기본 구현됨 (상세 조회만)

### 5. 완료 프로젝트 (Completed Projects) - `end`
- **기능**: 프로젝트 상세조회
  - 진행 완료된 프로젝트 리스트 표시
  - 완료된 프로젝트 상세 조회
- **현재 상태**: 
  - ✅ 상태 표시는 있음 (`end` 상태)
  - ❌ 필터 버튼 없음 (추가 필요)

## 구현 필요 사항

### 1. 완료 프로젝트 필터 추가
```typescript
// 필터 버튼에 추가 필요
<button
  onClick={() => setStatusFilter('end')}
  className={...}
>
  ✅ 완료 ({projects.filter(p => (p.counsel_status || p.status) === 'end').length})
</button>
```

### 2. 매칭중 상태에서 프로젝트 수정 기능
- 매칭중(`recruiting`) 상태의 프로젝트 카드에 "프로젝트 수정" 버튼 추가
- 수정 버튼 클릭 시 프로젝트 수정 페이지로 이동
- 수정 가능한 항목:
  - 프로젝트 서비스 이름
  - 기능
  - 기간
  - 비용
  - 기타 요구사항

### 3. 각 상태별 액션 버튼 정리
- **매칭중**: "프로젝트 수정" 버튼 + "상세 조회"
- **견적 도착**: "견적서 검토" 버튼 + "상세 조회" (현재 구현됨)
- **계약 진행**: "상세 조회"만
- **완료**: "상세 조회"만

## 데이터베이스 상태 값
- `pending`: 접수됨
- `recruiting`: 매칭중
- `estimate_received`: 견적 도착
- `contract_progress`: 계약 진행
- `end`: 완료

## API 엔드포인트 필요 사항
1. 프로젝트 수정 API (`PUT /api/counsel/:id`)
2. 완료 프로젝트 조회 (이미 `fetchAllCounsel`에 포함되어 있을 것으로 예상)
